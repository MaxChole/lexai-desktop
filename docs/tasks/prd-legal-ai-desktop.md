## 1. 介绍 / 概述

### 问题陈述

律师、法务和法学生在使用 AI 辅助法律工作时面临四大障碍：

1. **技术门槛高** — 配置 Claude Code CLI、安装插件、管理 Prompt 需要技术能力，大多数法律从业者无法自行完成。
2. **法律体系割裂** — 现有工具要么只支持中国法律体系（ZH fork），要么只支持美国/国际体系（anthropics 原版），跨境法务无法在同一工具内切换。
3. **工作流不连贯** — 没有统一的文档管理、案件库和历史会话，每次工作都要从零开始。
4. **隐私与合规风险** — 法律文件极度敏感，云端 API 调用存在数据外传风险；企业客户可能有数据不出境要求；出差、法庭、断网场景无法工作。

### 解决方案

**LexAI Desktop** 是一款 Electron 桌面应用，内置 Skill 运行时引擎，将 `anthropics/claude-for-legal`（国际）和 `CSlawyer1985/claude-for-legal-ZH`（中国）的全部 SKILL.md 和 Agent .md 文件解析并执行。

支持两种运行模式：
- **云端模式（联网）** — 平台统一管理多模型路由（Claude / DeepSeek / Kimi），按套餐分配模型质量，用户无需配置 API Key。
- **本地模式（可离线）** — 通过内置本地推理引擎 sidecar 下载并运行本地大模型（Qwen2.5 / Llama），所有数据留在本机，零网络依赖，满足隐私合规要求。默认使用应用内嵌 runtime，不要求用户预装 Ollama；若系统已安装 Ollama，可作为兼容 provider 接入。

## 2. 目标

- **G-1:** 上线后 6 个月内，月活付费用户 ≥ 500。
- **G-2:** 用户完成一次完整法律任务（合同审查/起草/风险分析）平均耗时 ≤ 15 分钟。
- **G-3:** 两个仓库的 Skill 覆盖率各自 100%，自研跨法域对照 Skill ≥ 6 个。
- **G-4:** 月续订率 ≥ 70%。
- **G-5:** 本地模式支持主流硬件（≥ 16GB RAM），7B 模型响应首 token ≤ 15 秒（CPU-only）。

**US-008: 下载并使用本地大模型（离线/隐私模式）**
作为对数据隐私有严格要求的企业法务，我想在应用内下载本地大模型并切换到本地模式，以便在断网环境或数据不出境场景下继续使用所有 Skill。

验收标准：
- 设置页有"本地模型"选项卡，显示可下载的模型列表（名称/大小/推荐场景）

- 本地模式下 Agent 定时任务仍可执行（无需联网）
- Verify in browser using dev-browser skill

## 5. 功能需求

### 核心平台

**FR-1:** 应用基于 Electron 28+ 构建，支持 macOS（Intel + Apple Silicon）和 Windows 10/11。

**FR-2:** 多模型路由层统一管理所有 AI 调用，支持云端和本地两类 provider：

**云端模式（联网）：**

| 套餐 | 默认模型 | 备选模型 | 说明 |
|------|---------|---------|------|
| 入门版 | DeepSeek-V3 | Kimi | 降低服务成本 |
| 专业版 | claude-sonnet-4-6 | DeepSeek-V3 | 质量与成本平衡 |
| 企业版 | claude-opus-4-6 | claude-sonnet-4-6 | 最高质量 |

**本地模式（离线/隐私）：**

初版仅提供一个本地模型（验证流程后迭代扩充）：

| 模型 | 文件格式 | 大小 | 场景 | 最低硬件 |
|------|---------|------|------|---------|
| Qwen2.5-7B-Instruct-Q4_K_M | GGUF 量化 | ~5 GB | 中英文法律任务，日常使用 | 16GB RAM |

用户可在专业版及以上手动切换云端备选模型；所有订阅用户均可使用本地模式。

**FR-3:** 用户认证：邮箱/密码 + Google OAuth，JWT 令牌 Electron 本地加密存储。

**FR-4:** 订阅三档：**入门版**（个人，Token 限额/月）、**专业版**（个人，更高限额 + Agent + 手动选模型）、**企业版**（多席位 + 管理后台 + 自定义配额）。**所有套餐均包含本地模式，本地模式不消耗 Token 配额。**

### Skill 运行时引擎

**FR-5:** SkillEngine 解析两个仓库的全部 `SKILL.md` 文件：
- 读取 YAML 前置元数据（name, description, argument-hint, user-invocable, model, tools）
- 将 Markdown 指令正文构建为 System Prompt
- 注入用户的 Practice Profile（对应原始仓库的 `CLAUDE.md` 配置）
- 按 jurisdiction 标签索引（CN / US / INT / CROSS）

**FR-6:** Slash Command 补全菜单按当前法律体系过滤 `user-invocable: true` 的 Skills。跨法域对照模式显示 `CROSS` Skills。

**FR-7:** 自研 6 个跨法域对照 Skill（jurisdiction: CROSS），并行调用 CN + US System Prompt，输出双栏对照：
1. 中美合同双体系审查
2. 中美数据合规对比（PIPL vs GDPR/CCPA）
3. 跨境并购尽职调查
4. 中美知识产权对比
5. 跨境劳动合规对比
6. 跨境争议解决策略

**FR-8:** Skills 更新：后台定期同步两个 GitHub 仓库，重启后加载最新版本。

### 文档与案件管理

**FR-9:** 案件 CRUD，每案件支持：标题、描述、标签、文档（PDF/Word/TXT，≤20MB）、会话历史。

**FR-10:** 文档 S3 AES-256 加密存储，`user_id` 隔离，不可跨用户访问。

**FR-11:** 会话历史持久化，支持关键词/日期/Skill 三维搜索。

### 定时 Agent

**FR-12:** 集成两仓库所有 headless agent（约 10 个），在 Agent 面板展示。

**FR-13:** Agent 结果通过 Electron Notification API + 应用内消息中心推送。

**FR-14:** Agent Token 消耗计入套餐月度配额。

### 引用与合规

**FR-15:** AI 输出保留引用安全标记：CN 体系用 `[需验证]`，EN 体系用 `[verify]`，不可被自动移除。

**FR-16:** 每条 AI 响应后固定显示免责声明："本内容由 AI 生成，仅供参考，不构成正式法律意见。"

### 本地大模型（本地推理引擎集成）

**FR-17:** 应用以 sidecar 进程方式内置本地推理引擎。应用启动时检测 embedded runtime 是否运行，若未运行则自动启动；应用退出时自动关闭本地推理进程。用户无需手动安装 Ollama。若系统已安装 Ollama，可作为兼容 provider 接入，但不是功能前提。

**FR-18:** 模型管理 UI（设置页 → 本地模型选项卡）提供：
- 可下载模型列表（模型名、大小、推荐场景、推荐硬件）
- 每个模型的下载按钮，下载时显示实时进度条（已下载/总大小、速度、预计剩余时间）
- 支持暂停/恢复下载
- 已安装模型显示"删除"按钮（释放磁盘空间）
- 下载前检测硬件（RAM / 可用磁盘），不满足推荐配置时显示黄色警告（不阻止下载）

**FR-19:** 顶部工具栏显示当前模式（云端 / 本地：模型名），点击可切换。切换到本地模式的前置条件：至少已安装一个本地模型。

**FR-20:** 本地模式下的数据隔离：
- AI 调用默认走 embedded runtime 的本地 REST 接口，不经过后端服务器
- 若用户启用了兼容模式，可改走本机 Ollama OpenAI-compatible 接口
- 案件、文档、会话历史存储在本地 SQLite（`userData/local.db`），不同步到云端
- 上传的文件存储在本地文件系统（`userData/documents/`），不上传 S3
- 本地模式下不记录 Token 用量到云端（仅本地统计）

**FR-21:** 云端模式与本地模式的数据可选择性同步：用户在切换回云端模式时，可选择将本地会话历史和案件上传到云端账号（需网络连接）。文档默认不同步（避免隐私数据意外上云），需用户主动勾选每个文件。

## 6. 非目标

- 不做 Web 版（初版仅 Electron）
- 不做移动端
- 不做人工律师对接
- 不自建法律数据库
- 不开放用户自定义上传 Skills
- 不做语音交互
- 不做多人实时协作编辑
- **不自行训练或微调模型** — 本地模式只下载开源预训练模型，不做 Fine-tuning
- **不支持用户任意导入第三方模型文件** — 初版只支持应用内定义的白名单模型列表；兼容 Ollama 时也仅支持白名单模型映射

```
               │  └──────────┘ └──────┘  │  └─────────────────┘ │
               └─────────────────────────┴──────────────────────┘
```

### Skill 运行时流程

```
用户输入 "/commercial-legal:review"
    │
    ▼
SkillEngine.resolve("commercial-legal", "review")
    └─ 读取 SKILL.md → 解析 frontmatter + 正文
    └─ 注入用户 Practice Profile（CLAUDE.md 内容）
    └─ 构建 System Prompt（含 Prompt Cache 标记）
    │
    ▼
ModelRouter.call(systemPrompt, userMessage, userPlan)
    └─ 按套餐选择模型
    └─ 调用对应 API（含 cache_control）
    │
    ▼
响应流式返回 → 前端渲染 → [需验证] 标记高亮
```

### Prompt Caching 策略

Skill System Prompt（通常 2000-8000 tokens）作为缓存前缀，对同一 Skill 的重复调用命中缓存，预计节省 60-80% 输入 Token 费用。
